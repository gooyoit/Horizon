## [SGLang v0.5.11 升级至 CUDA 13，将投机解码 V2 设为默认](https://github.com/sgl-project/sglang/releases/tag/v0.5.11) ⭐️ 8.0/10

SGLang v0.5.11 通过升级至 CUDA 13 和 PyTorch 2.11 实现了核心基础设施的现代化，将投机解码 V2 设为默认以加速推理，并为包括 Gemma 4、GLM-5.1、Qwen3.6 和 Kimi-K2.6 在内的众多新发布的前沿 AI 模型提供了首日部署支持。 作为大语言模型的关键高性能推理引擎，SGLang 的基础设施升级和推理优化直接推动了最先进 AI 部署能力的发展，使生产团队能够从模型发布首日起就更快、更高效地部署最新的前沿模型。 值得注意的技术新增包括用于预填充/解码分离部署的解码端 Radix Cache 前缀缓存、对 DeepSeek-V3 和 Kimi-K2 等最大 MLA 架构 MoE 模型的 LoRA 支持、社区贡献的 FA3 和 DFLASH 投机解码内核（支持 AMD ROCm），以及允许独立调节 MoE 和注意力并行度的上下文并行增强。

github · sgl-project/sglang · May 5, 21:28

**背景**: SGLang 是一个高性能推理引擎，旨在大规模高效运行大语言模型推理。投机解码是一种推理优化技术，通过同时预测和验证多个 token 来加速大语言模型，在不降低输出质量的前提下减少延迟。预填充/解码分离将大语言模型推理的两个阶段——预填充（处理提示词）和解码（生成 token）——分配到不同硬件上，以提高吞吐量并降低尾延迟。RadixCache 是 SGLang 的前缀缓存机制，通过 Radix Tree 实现高效查找，在共享相同提示词前缀的请求之间复用键值（KV）缓存。
