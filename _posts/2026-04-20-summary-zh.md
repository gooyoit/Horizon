## [Claude Token Counter 新增模型对比功能](https://simonwillison.net/2026/Apr/20/claude-token-counts/#atom-everything) ⭐️ 8.0/10

Simon Willison 更新了他的 Claude Token Counter 工具，支持在多个 Claude 模型（包括 Opus 4.7、Opus 4.6、Sonnet 4.6 和 Haiku 4.5）之间进行并排的 token 数量对比。此次更新突显了 Claude Opus 4.7 引入的新 tokenizer 所带来的变化：与 Opus 4.6 相比，文本 token 数量最多增加 1.46 倍，图像则高达 3.01 倍。 这一点很重要，因为 tokenizer 的变化直接影响成本和性能——尽管 Opus 4.7 的每 token 定价未变，但更高的 token 数量意味着用户可能多支付约 40% 的费用。开发者和研究人员需要准确的 token 估算来控制预算、优化提示词，并理解不同版本模型的行为差异。 该工具使用 Anthropic 官方的 token 计数 API，并确认在当前所有模型中，只有 Opus 4.7 更换了 tokenizer，其余模型与 Opus 4.6 共用相同的 tokenizer。由于 Opus 4.7 增强了对高分辨率图像的支持（最高约 375 万像素），图像输入下的 token 膨胀现象更为显著。

rss · Simon Willison · Apr 20, 00:50

**背景**: Tokenization 是大语言模型将文本（有时也包括图像）转换为数值化 token 以进行处理的过程。token 数量会影响计算成本和上下文窗口的使用。Anthropic 的 Claude 模型按输入和输出的 token 数量计费，因此 tokenizer 的变更会直接影响用户支出。Opus 4.7 是 Anthropic 最新发布的旗舰模型，在推理、编程和多模态理解方面均有提升。

## [无头服务推动个人 AI 代理兴起](https://simonwillison.net/2026/Apr/19/headless-everything/#atom-everything) ⭐️ 8.0/10

Matt Webb 和 Salesforce 首席执行官 Marc Benioff 认为，无头化、API 优先的服务正变得对个人 AI 代理高效与平台交互至关重要。Salesforce 已推出“Headless 360”，通过 API、MCP 和 CLI 公开其整个平台（包括 Agentforce 和 Slack），供 AI 代理直接访问。 这一转变使 AI 代理能够绕过图形用户界面，更可靠、更快速地运行，从根本上改变用户与软件的交互方式。它还威胁到传统的按用户计费的 SaaS 定价模式，因为 AI 代理而非人类将成为服务的主要使用者。 Salesforce 的 Headless 360 使用了 Anthropic 于 2024 年推出的开放标准 Model Context Protocol（MCP），让 AI 代理能在 Slack 和语音界面等平台间访问数据和工作流。此举呼应了 2010 年代初的 API 热潮，但如今的驱动力是 AI 代理的需求，而不仅仅是人类开发者。

rss · Simon Willison · Apr 19, 21:46

**背景**: 在软件架构中，“无头”服务指仅通过 API 暴露功能、不带内置用户界面的后端系统，与用户通过图形界面（GUI）交互的传统应用形成对比。Model Context Protocol（MCP）是一种开放标准，允许 AI 模型安全地连接外部工具、数据源和工作流，充当 AI 代理的通用集成层。
