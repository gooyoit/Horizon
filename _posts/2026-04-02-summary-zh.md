## [GitHub 现非官方仓库，从 npm sourcemap 还原 Claude Code 源码](https://t.me/zaihuapd/40641) ⭐️ 9.0/10

一个名为 claude-code-sourcemap 的 GitHub 仓库声称，通过公开 npm 包@anthropic-ai/claude-code 中 cli.js.map sourcemap 文件的 sourcesContent 字段，还原出 Anthropic 公司 Claude Code 2.1.88 版本的 4756 个源代码文件。 此次事件暴露了领先 AI 公司专有编程助手的内部工具代码，引发了对知识产权保护、软件供应链安全以及在生产构建中附带 sourcemap 可能带来意外后果的担忧。同时也加剧了前沿 AI 开发中透明度与保密之间的争论。 此次还原依赖于 sourcemap 中的 sourcesContent 字段，该字段直接嵌入了原始源代码；恢复的文件包含 1884 个 TypeScript（.ts/.tsx）文件。这种方法之所以可行，是因为 Anthropic 在发布 npm 包时包含了完整的源代码内容——这在 JavaScript/TypeScript 项目中虽常见但存在风险。

telegram · zaihuapd · Apr 1, 08:07

**背景**: 在 JavaScript 和 TypeScript 开发中，sourcemap 文件（如.map 文件）用于将压缩或转译后的代码映射回原始源代码以便调试。当包含'sourcesContent'字段时，它会将完整的原始源代码直接嵌入 sourcemap 中。如果此类 sourcemap 被公开发布，就可能无意中泄露专有源代码，本次 Claude Code 事件正是如此。
