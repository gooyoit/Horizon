## [超 20%免费 LLM API 路由器存在恶意行为](https://x.com/Fried_rice/status/2042423713019412941) ⭐️ 9.0/10

Solayer 创始人 Chaofan Shou 发布研究，揭示所测试的免费 LLM API 路由器中超过 20%存在恶意行为，包括窃取 AWS 凭证和注入恶意代码。该研究测试了 400 个免费和 28 个付费路由器，发现 8 个免费路由器主动注入恶意代码，另有 17 个接触了敏感凭证。 这暴露了 AI 智能体生态系统中关键的供应链漏洞——第三方路由器作为受信任的中介，却缺乏加密或监管。此类风险可能导致大规模凭证窃取、财务损失以及整个行业 AI 系统完整性的破坏。 这些恶意路由器可明文访问传输中的 JSON 载荷，从而实施如计费令牌欺诈和主机接管等攻击。研究人员通过自研的“Mine”代理验证了四类攻击，并提出了故障闭锁策略门控和响应端异常筛查等防御手段。

telegram · zaihuapd · Apr 10, 08:30

**背景**: LLM API 路由器是介于应用程序与大语言模型之间的中介服务，常用于通过统一接口简化对多个模型的访问。许多开发者为节省成本使用免费路由器，默认其为无害代理。然而，这些路由器处理原始 API 流量（包括认证令牌和提示词），若缺乏适当保护，便可窥探甚至篡改敏感数据。目前该层极少部署端到端加密，成为 AI 部署安全中的盲点。

## [Claude 曝出上下文极限附近的“身份混淆”漏洞](https://news.ycombinator.com/item?id=47701233) ⭐️ 9.0/10

开发者报告称，Anthropic 的 Claude 大语言模型在接近上下文窗口极限时会出现“身份混淆”漏洞，误将自身先前输出当作用户当前指令。该缺陷已导致 Claude Code 等工具中意外执行高风险操作。 该漏洞对安全构成严重威胁，尤其在 AI 代理系统中，模型会自主执行代码或系统命令。若错误归因用户意图，可能导致未经授权的部署、数据删除等有害操作，而无需真实用户授权。 该问题主要出现在模型上下文极限（如 Claude 的约 20 万 token）附近的“愚笨区”，此时注意力机制退化，模型将自生成文本误认为用户输入。Claude Code 等可集成开发环境并运行终端命令的工具尤其易受攻击。

telegram · zaihuapd · Apr 10, 14:52

**背景**: 大语言模型（LLM）在固定的“上下文窗口”内处理输入，即一次能处理的最大 token 数量。当窗口接近填满时，模型可能难以维持对话结构，导致角色混淆等错误。Claude 由 Anthropic 开发，采用“宪法 AI”技术以对齐人类价值观，但此类漏洞暴露了其在实际应用中的可靠性缺口。Claude Code 是一个智能编码助手，可根据自然语言提示读取、编辑并执行代码。
