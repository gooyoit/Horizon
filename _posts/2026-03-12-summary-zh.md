## [Temporal API 历经九年终修复 JavaScript 时间处理问题](https://bloomberg.github.io/js-blog/post/temporal/) ⭐️ 9.0/10

历经九年开发，JavaScript 的现代且健壮的时间处理系统 Temporal API 终于落地。它通过明确且安全的语义（如时间点、日历日期、时区和持续时间）取代了容易出错的传统 Date 对象。 Temporal 消除了困扰 JavaScript 应用数十年的时区、夏令时和模糊日期解析等常见但代价高昂的 bug。这一改进提升了全球应用的可靠性，并减少了因细微时间错误导致的生产事故。 Temporal 明确区分绝对时间（Instant）与人类可读的日历/时区概念（如 PlainDateTime、ZonedDateTime），防止误用。然而，Temporal 对象无法直接序列化为纯 JSON，可能在某些客户端-服务器架构中增加数据传输复杂度。

hackernews · robpalmer · Mar 11, 15:35

**背景**: JavaScript 原始的 Date 对象源自早期 Java，用单一类型同时表示时间点和日历日期，导致在时区和夏令时等问题上频繁出现歧义和 bug。Temporal 提案由 TC39（JavaScript 标准委员会）提出，旨在借鉴 Python、C# 等语言的经验，通过类型安全、不可变且全面的设计来修复这些缺陷。
