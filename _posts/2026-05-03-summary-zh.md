## [VS Code 默认添加“Co-Authored-by Copilot”](https://github.com/microsoft/vscode/pull/310226) ⭐️ 8.0/10

微软的 VS Code 现在默认在 Git 提交信息中自动插入“Co-Authored-by: GitHub Copilot”，即使只是自动补全等微小的 AI 辅助编辑也会触发。此更改在 VS Code 1.118 版本中引入，除非用户手动关闭，否则会影响所有用户。 此举将营销与技术诚信混为一谈，损害了开发者对工具的信任，因为 Git 提交是代码作者身份的法律和历史记录。它引发了关于用户同意、透明度以及 AI 在软件开发工作流中伦理使用的严重担忧。 该功能采用“退出式”而非“加入式”设计，尽管运行时回退逻辑仍默认为“关闭”，但配置架构已被改为“全部”，导致行为不一致。据报道，GitHub Copilot 本身曾在该拉取请求中指出这一不一致问题，但被忽视了。

## [NASA 阿尔忒弥斯二号任务用激光从月球传回 484GB 数据](https://dailygalaxy.com/2026/05/nasa-just-beamed-484-gigabytes-from-moon/) ⭐️ 7.0/10

在阿尔忒弥斯二号任务中，NASA 利用由 MIT 林肯实验室开发的猎户座阿尔忒弥斯二号光学通信系统（O2O），成功从月球向地球传输了 484GB 数据，下行速率高达 260 Mbps。 这一突破为未来的月球和火星任务提供了可行的高带宽通信基础设施，能够近乎实时地传输高清图像，并支持依赖大量数据的 AI 驱动自主系统。相比带宽受限的传统射频系统，这是重大飞跃。 O2O 系统使用红外激光而非无线电波，峰值速率达 260 Mbps；位于加州（喷气推进实验室）、新墨西哥州（白沙综合设施）和澳大利亚（斯特罗姆洛山天文台）的地面站接收了数据，其中一站不到一小时就下载了 26 GB。

telegram · @zaihuapd · May 3, 00:50

**背景**: 空间光通信利用激光在远距离传输数据，相比传统射频系统可提供高得多的带宽。NASA 已研发该技术十余年，早期如 2013 年的月球激光通信演示项目已验证其可行性。O2O 系统在此基础上发展，旨在为载人深空任务提供可靠高速的通信链路。与射频不同，激光通信需要精确对准，但能在不增加功耗、质量和体积的前提下实现更快的数据传输。
