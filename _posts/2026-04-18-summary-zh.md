## [谷歌拟在五角大楼机密环境部署 TPU 芯片](https://www.tomshardware.com/tech-industry/artificial-intelligence/google-and-pentagon-in-talks-to-run-tpus-inside-classified-environments) ⭐️ 9.0/10

谷歌正与美国国防部谈判，计划在其机密环境中部署自研张量处理单元（TPU）和 GPU 机架，以大规模运行 Gemini 人工智能模型用于国防任务。这将是谷歌 TPU 基础设施首次进入此类高安全级别的政府设施。 此举使谷歌能直接与 AWS 和微软竞争利润丰厚的美国国防云市场，同时推动前沿 AI 在国家安全领域的应用。此外，它还为军事 AI 使用设定了重要伦理先例，例如禁止全自动武器和国内大规模监控。 尽管谷歌分布式云已于 2025 年 5 月获得影响级别 6（IL6）授权，可处理最高机密数据，但其在机密边界内仍缺乏支撑大规模 AI 工作负载的硬件基础设施。拟议协议包含与 OpenAI 此前达成的国防部协议类似的使用限制条款。

## [Claude 4.7 新分词器在定价不变下推高成本](https://www.claudecodecamp.com/p/i-measured-claude-4-7-s-new-tokenizer-here-s-what-it-costs-you) ⭐️ 8.0/10

Anthropic 最新模型 Claude Opus 4.7 采用了更新的分词器，将相同输入文本映射为比前代多 1.0 至 1.35 倍的 token 数量，尽管每 token 价格维持不变（输入$5/百万 token，输出$25/百万 token），实际推理成本却因此上升。 这一变化凸显了前沿 AI 开发中的关键权衡：微小的性能提升可能带来不成比例的更高运营成本，挑战了模型扩展的经济可持续性。这也影响依赖精确成本预测来构建 AI 应用的开发者和企业。 分词器更新在不改变 API 定价层级的情况下增加了 token 数量，意味着用户为相同长度的输入支付更多费用。此外，Claude 4.7 默认启用了更高的“思考努力”级别，生成更多输出 token，进一步加剧成本上升。

hackernews · aray07 · Apr 17, 15:29

**背景**: 分词器将文本转换为大语言模型处理的数值单元（token）。token 数量直接决定 Anthropic 等服务商的 API 成本。更细粒度的分词器虽可提升模型理解能力，但通常会使相同文本产生更多 token。近期 GPT 和 Claude 系列等大模型的迭代中，分词器不断优化，对性能和成本效率均产生影响。
